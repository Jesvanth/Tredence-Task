01 — Problem Statement

Why Self-Pruning?

Modern deep networks carry millions of redundant parameters. We can do better.

💾

The Cost Problem

Large neural networks require enormous memory and compute. A standard feed-forward network for CIFAR-10 holds 3.7M+ parameters, most of which contribute little to accuracy.

✂️

Traditional Pruning — Too Late

Post-training pruning removes weights after a model is already trained. This is a two-step process and cannot adapt dynamically. The network has no chance to learn to be sparse.

🧠

Our Approach — Learn to Prune

We give each weight a learnable gate ∈ (0, 1). When the optimizer finds a weight unnecessary, it drives that gate toward zero — permanently silencing it during training.

⚙️

The Mechanism

Effective Weight = W × sigmoid(gate_score)

As gate → 0, the weight is effectively removed. Gradients flow through both the weight and the gate score, making this fully end-to-end trainable.

Step 1
Gate Generation
gate = σ(gate_score)

Each weight has a learnable score. Sigmoid maps it to (0, 1) — a soft switch.

Step 2
Pruned Weight
W_pruned = W × gate

Element-wise multiplication. When gate ≈ 0, the weight is effectively dead.

Step 3
Total Loss
L = CE + λ × Σ|gates|

Cross-entropy trains for accuracy. L1 gate penalty drives unnecessary gates to zero.

Result
Sparsity Loss
L_spar = Σ gates (all layers)

Sum of all gate values across layers. Minimized by optimizer → sparse network.

05 — Gate Analysis

Gate Distribution

After training, gate values polarize — most collapse near 0, a minority stay near 1. This bimodal pattern is the hallmark of successful self-pruning.

~73% gates
collapsed near 0
Redundant connections removed
~27% gates
remain near 1
Essential connections preserved

What This Means

Bimodal Distribution

Gates don't hover in the middle — they commit. Either a connection matters (gate ≈ 1) or it doesn't (gate ≈ 0). This is exactly what we want.

L1 Drives Zero-Collapse

The L1 penalty creates a constant gradient pull toward zero regardless of current gate value. Unlike L2, small gates still feel the full push.

No Manual Engineering

The network discovers which connections matter entirely on its own. No human-defined importance scores, no structured masks — purely data-driven.

Real-time Adaptability

Because gates are learned jointly with weights, the network can compensate — redistributing importance to surviving connections as pruning progresses.

06 — Theory

Why L1 Encourages Sparsity

The mathematical intuition behind why L1 — not L2 — is the right tool for pushing gates to zero.

01

Constant Gradient Pressure

∂|x|/∂x = sign(x)

The gradient of L1 is ±1 regardless of magnitude. Even a gate at 0.001 feels the same push toward zero as a gate at 0.5. This is why L1 produces true zeros.

02

L2 Fails Here

∂x²/∂x = 2x → 0 as x→0

L2's gradient shrinks to zero as the weight shrinks. Small gates feel almost no pressure — they never fully collapse, just become very small. No true sparsity.

03

Optimizer Mechanics

gate_new = gate - lr × (∂CE/∂g + λ·sign(g))

AdamW steps push gates toward zero via the λ·sign(gate) term at every update. Over 30 epochs, unnecessary gates accumulate pressure until they collapse.

04

The Geometric View

L1 regularization corresponds to an L1-ball constraint — a diamond in weight space. Its corners lie on the axes, making sparse solutions (many zeros) the most likely minimum of the constrained problem.

Custom Neural Layer from Scratch

PrunableLinear extends nn.Module from scratch — no wrappers. Implements custom forward pass, gate extraction, sparsity computation, and extra_repr for debugging.

class PrunableLinear(nn.Module):
    def forward(self, x):
        gates = torch.sigmoid(self.gate_scores)
        W = self.weight * gates
        return F.linear(x, W, self.bias)
🛡️

Gradient-Safe Gating Mechanism

Gate scores remain in the compute graph during loss calculation. Sigmoid is differentiable everywhere — gradients flow cleanly to both weights and gate scores simultaneously.

def sparsity_loss(self):
    total = torch.tensor(0.0, requires_grad=True)
    for layer in self.prunable_layers():
        gates = torch.sigmoid(layer.gate_scores)
        total = total + gates.abs().sum()
    return total
